# Catapult Creator Bot

Automatically creates tokens using Catapult creator package daily limits. The bot logs in to your account, checks available daily credits, and spreads token creation evenly across the day.

## Prerequisites

- Node.js >= 20
- An EVM wallet with a Catapult account
- Active creator package(s) purchased on the account

## Quick Start

### 1. Install dependencies

```bash
npm install
```

### 2. Configure

```bash
cp .env.example .env
```

Edit `.env` and set your EVM private key:

```env
EVM_PRIVATE_KEY=0xYOUR_PRIVATE_KEY_HERE
```

### 3. Run

```bash
# Development (with hot reload)
npm run dev

# Production
npm run build
npm start
```

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `EVM_PRIVATE_KEY` | *required* | EVM wallet private key (0x-prefixed) |
| `API_URL` | `https://api.catapult.trade/graphql` | Catapult API endpoint |
| `TOKEN_RANK` | `Public` | Token rank (`Public` or `Private`). Public required for package credits |
| `TOKEN_MODE` | `Mayhem` | Token mode: `Slow`, `Fast`, `Flash`, `Crack`, `Mayhem` |
| `TOKEN_PRICE_MODE` | `FiveHundred` | Price mode: `One`, `Ten`, `Hundred`, `FiveHundred` |
| `SPREAD_START_HOUR` | `0` | Hour (0-23) to start creating tokens |
| `SPREAD_END_HOUR` | `23` | Hour (0-23) to finish creating tokens by |
| `TIMEZONE` | `UTC` | Timezone for scheduling |
| `LOG_LEVEL` | `info` | Log level: `fatal`, `error`, `warn`, `info`, `debug`, `trace` |

### Token Cost per Creation

| Mode | Public | Private |
|------|--------|---------|
| Slow | 12 USDT | 1 USDT |
| Fast | 10 USDT | 1 USDT |
| Flash | 8 USDT | 1 USDT |
| Crack | 5 USDT | 1 USDT |
| Mayhem | 1 USDT | 1 USDT |

Cheaper modes = more tokens created per daily budget.

## How It Works

1. **Login**: Signs a challenge message with your EVM private key
2. **Check limits**: Queries your daily creator package credit balance
3. **Calculate**: Determines how many tokens can be created with remaining credits
4. **Spread**: Evenly distributes token creation across `SPREAD_START_HOUR` to `SPREAD_END_HOUR`
5. **Create**: Calls the randomized token preset API for each token (name, symbol, avatar are auto-generated)
6. **Repeat**: At `SPREAD_START_HOUR` each day, starts a new cycle

If the bot starts late in the day and there isn't enough time to spread evenly, it creates tokens as quickly as possible (5-second intervals).

## Docker Deployment

### Build and run locally

```bash
docker compose up -d
```

### Deploy to AWS (ECS Fargate)

1. Push image to ECR:
```bash
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin <account-id>.dkr.ecr.us-east-1.amazonaws.com
docker build -t catapult-creator-bot .
docker tag catapult-creator-bot:latest <account-id>.dkr.ecr.us-east-1.amazonaws.com/catapult-creator-bot:latest
docker push <account-id>.dkr.ecr.us-east-1.amazonaws.com/catapult-creator-bot:latest
```

2. Create ECS task definition with the image and environment variables
3. Run as a Fargate service (1 task, no load balancer needed)

### Deploy to EC2

```bash
# On the EC2 instance
git clone <repo-url>
cd catapult-creator-bot
cp .env.example .env
# Edit .env with your credentials
docker compose up -d
```

## Important Notes

- The bot does **not** make deposits or purchase packages - only uses existing credits
- Only `Public` rank tokens can use creator package credits
- The bot auto-refreshes authentication if the JWT token expires
- All token names, symbols, and avatars are randomly generated by the Catapult API
